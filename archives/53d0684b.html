Linux目录结构

Linux目录结构采用层级式树状目录结构，此结构中最上层的是根目录/。

具体目录

/bin：存放经常使用的命令

/sbin：存放系统管理员使用的系统管理程序

/home：存放普通用户的主目录的目录

/root：超级权限者的用户主目录

/lib：动态链接共享库

/lost+found：系统非法关机后，存放文件

/etc：存放配置文件和子目录

/usr：用户的应用程序和文件存放在此目录下

/boot：存放启动Linux时使用的核心文件，包括连接文件和镜像文件

/tmp：存放临时文件

dev：把所有的硬件用文件的形式存储

/media：U盘、光驱等识别后，会把识别的设备挂载到此目录下。

/mnt：可以将外部的存储挂载到/mt

/opt：给主机额外安装软件所存放的目录

/usr/local：给主机额外安装软件所存放的目录

/var：存放经常修改的目录，例如日志文件。

进程管理类命令

ps

选项 功能
-a 列出带有终端的所有用户的进程
-x 列出当前用户的所有进程，包括没有终端的进程
-u 列出某个用户关联的所有进程
-e 列出所有进程
-f 显示完整格式的进程列表

ps -aux 显示信息说明，想查看进程CPU占用率和内存占用率，用ps -aux

  • USER：该进程是由哪个用户产生的

  • PID：进程号

  • %CPU：进程占用CPU资源百分比

  • %MEM：进程占用内存百分比

  • VSZ：进程占用虚拟内存的大小，单位KB

  • RSS：进程占用内存的大小，单位KB

  • TTY：进程是在哪个终端中运行的 ，tty1 是图形化终端， tty2-tty6 是本地的字符界面终端。pts/0-255 代表虚拟终端。

  • STAT：进程状态，常见的状态有：R：运行状态、S：睡眠状态、T：暂停状态、 Z：僵尸状态、s：包含子进程、l：多线程、+：前台显示

  • START：进程的启动时间

  • TIME：进程占用CPU运算的时间

  • COMMAND：产生此进程的命令名

ps -ef 显示信息说明，想查看进程的父进程ID，用ps -ef

  • UID：该进程是由哪个用户产生的
  • PID：进程号
  • PPID：父进程进程号
  • C：CPU 用于计算执行优先级的因子。数值越大，表明进程是 CPU 密集型运算， 执行优先级会降低；数值越小，表明进程是 I/O 密集型运算，执行优先级会提高
  • STIME：进程的启动时间
  • TTY：进程是在哪个终端中运行的 ，tty1 是图形化终端， tty2-tty6 是本地的字符界面终端。pts/0-255 代表虚拟终端。
  • CMD：产生此进程的命令名

软件包管理

RPM

RPM（RedHat Package Manager），RedHat软件包管理工具，类似windows里面的setup.exe 是Linux这系列操作系统里面的打包安装工具。

查询命令：

卸载命令：

选项 功能
-e 卸载软件包
–nodeps 卸载软件时，不检查依赖。

安装命令：

选项 功能
-i install，安装
-v –verbose，显示详细信息
-h –hash，进度条
–nodeps 安装前不检查依赖

YUM

YUM（全称为 Yellow dog Updater, Modified）是一个在 Fedora 和 RedHat 以及 CentOS 中的 Shell 前端软件包管理器。基于 RPM 包管理，能够从指定的服务器自动下载 RPM 包 并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软件包，无须繁琐地一次 次下载、安装。

选项 功能
-y 对所有提问都回答“yes”
参数 功能
install 安装 rpm 软件
update 更新 rpm 软件
check-update 检查是否有可用的更新 rpm 软件
remove 删除指定的 rpm 软件
list 显示软件包信息
clean 清理 yum 过期的缓存
deplist 显示 yum 软件包的所有依赖关
